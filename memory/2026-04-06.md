
## Signal Checklist + Entry Tolerance Fix (03:35-06:10 UTC)

### Signal Checklist Feature (commit 901c80c)
- Added `/api/signal_checklist` endpoint in dashboard/app.py
- Two checklist sections: Pending Signals + Recent Trades
- Criteria evaluated per signal:
  1. SL Direction (correct side of entry)
  2. SL Gap % (≥ min_sl_gap_pct from config)
  3. Risk:Reward (≥ min_risk_reward from config)
  4. Confidence (≥ min_confidence, non-critical when 0/not stored)
  5. Zone Strength (N/A if not stored in pending_signals table)
  6. Entry Tolerance (actual gate from main.py, entry_tolerance_pct 0.3%)
  7. Entry Reachable (loose 3% proximity, informational)
  8. Entry Setup Valid (directional bias check)
- GO verdict: all critical criteria must pass
- Recent trades: uses get_closed_trades() (raw SQL had DataStore path issues)
- Verdict: SL Dir + SL Gap + R:R + Confidence (when >0) + Entry Tolerance = GO

### Bug Fixes During Checklist
- R:R was 0.00 for pending signals (pending_signals table doesn't store risk_reward) — fixed by calculating from entry/SL/TP
- Recent trades returned 0 (raw SQL with DataStore path mismatch) — fixed by using store.get_closed_trades()
- Zone Strength shows "N/A" when not stored (pending_signals table lacks column)
- Confidence shows "N/A" when 0 (trades table doesn't store confidence)
